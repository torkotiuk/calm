2. 2. To concat arrays

  • To concut arrays we may use old syntax:
    const firstArray = [10, 20, 30];
    const twoArrays = firstArray.concat([1, 2, 3]); //[10, 20, 30, 1, 2, 3]

  • new syntax with ... operation is const twoArrays = [...firstArray, 1, 2, 3];

2. 3. To create new array from couple another

  • Old syntax const newArr = Object.assign({}, {x: 1, y: 2}, {x: 5; z: 8}); //{x: 5, y: 2; z: 8}
  • New syntax: >>>>> const newArr = { ...{x: 1, y: 2}, ...{x: 5; z: 8}}; //{x: 5, y: 2; z: 8}

    Also in new syntax we can add new elements, f.e. w: 100, z: 500 somewhere const newArr = { ...{x: 1, y: 2}, w: 100, ...{x: 5, z: 8}, z: 500}; //{x: 5, y: 2; w: 100; z: 500, }

5. 2. Closure

Closure means that an inner function always has access to the vars and parameters of its outer function, even after the outer function has returned.

In other words - A closure is the combination of a function bundled together (enclosed) with references to its surrounding state (the lexical environment).

5. 3. This

The JavaScript this keyword refers to the object it belongs to.

In other words, every javascript function while executing has a reference to its current execution context, called this. Execution context means here is how the function is called.

To understand this keyword, only we need to know how, when and from where the function is called, does not matter how and where function is declared or defined.

Details about 'this'

By DEFAULT our without context. If we 'use strict' mode, our 'this' return 'undefined', but without strict mode our 'this' return 'window' (it's a tab in browser)

How to use a parcel template

  • Create own repository on github and clone to your pc
  • Copy all files from parcel-project-template-master to your clonned for pc repository (or from the original parcel-project-template https://github.com/parcel-project-template )
  • do one time for one project npm install
  • to create neccesary files for github pages do npm run build
  • change in package.json next lines
    • change username and name of project to yours --- "homepage": "https://YOUR-USERNAMA.github.io/NAME-REPOSITORY"
    • also change name of the project in "predeploy" --- "predeploy": "npm run build -- --public-url /NAME-REPOSITORY/"
  • do npm run deploy to create github pages in your remotely rep
